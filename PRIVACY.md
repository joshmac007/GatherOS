# GatherOS Privacy Policy

_Last updated: April 29, 2026_

GatherOS is a local-first desktop app for saving, browsing, and organizing visual references. This page explains what data we collect, where it lives, and what's done with it. The short version: **almost nothing leaves your machine, and what does is sent on your direct instruction.**

If anything below is unclear, email us at [hello@designjoy.co](mailto:hello@designjoy.co).

---

## What we don't do

- We do not run servers that collect, store, or analyze your data.
- We do not show ads or share data with advertisers.
- We do not track your usage, capture analytics, or report telemetry.
- We do not require you to create an account or sign in.

## Where your library lives

Every save, collection, tag, note, and thumbnail you create is stored on your computer in macOS's standard application data directory:

```
~/Library/Application Support/GatherOS/
```

Nothing in that folder is uploaded anywhere by GatherOS. You can browse it, back it up, or delete it directly in Finder. The app's **Erase Library** action in Settings removes everything in this folder; **Export Library as Zip** writes a portable copy you can move to another machine.

## OpenAI API key

GatherOS supports optional AI features (auto-naming, visual search, auto-tagging, image prompts). To use them you supply your own OpenAI API key.

- Your key is stored using the macOS Keychain (via Electron's `safeStorage`), encrypted by the operating system.
- The key never leaves your machine except to authenticate requests directly to `api.openai.com`.
- We do not see, log, transmit, or store your key on any server we control. We don't have a server.
- You can clear the key at any time in **Settings → OpenAI → Clear key**, or revoke it from your OpenAI dashboard.

## What gets sent to OpenAI

When AI features are enabled, GatherOS sends image data and text to OpenAI's API on your behalf. The traffic is between your machine and OpenAI; we are not in the middle.

What is sent depends on which features you've turned on:

- **Auto-name new uploads** sends each newly saved image to OpenAI's vision API for a short title.
- **Visual search** sends a brief textual description of each image (generated by the same vision call) to OpenAI's embeddings API so the search bar can match by meaning.
- **Auto-tag** and **Image prompts** each send a single image to OpenAI's vision API when you click the corresponding button.

OpenAI handles this data under their own [terms of use](https://openai.com/policies/terms-of-use) and [privacy policy](https://openai.com/policies/privacy-policy). If you'd rather not transmit anything to OpenAI, leave the AI features off (toggle them in **Settings → AI features**), and no calls are made.

## Auto-updates

GatherOS checks for new releases via GitHub Releases. The check is an HTTPS request to `github.com` for a metadata file (`latest-mac.yml`) and, when an update is available, the downloaded build itself. GitHub may log standard HTTP request information (IP address, user agent) the same way it does for any download — see [GitHub's privacy statement](https://docs.github.com/en/site-policy/privacy-policies/github-general-privacy-statement). GatherOS doesn't add anything to those requests beyond the version metadata electron-updater needs to do its job.

## Image storage

When you save an image, GatherOS writes the original file and a thumbnail into the application data directory above. Files dropped in by URL are downloaded directly from the source by your machine, processed locally, and saved. We don't proxy or copy them through any server.

## Crash and error reporting

GatherOS does not currently collect crash reports or error logs automatically. If you hit a bug, the most helpful thing is to email us a description (and a screenshot if you can) at [hello@designjoy.co](mailto:hello@designjoy.co). If we ever introduce optional crash reporting in the future, it will be opt-in and disclosed clearly in this document.

## Children

GatherOS is not directed at children under 13. If you are a parent or guardian and believe your child has used GatherOS, contact us at [hello@designjoy.co](mailto:hello@designjoy.co).

## Changes to this policy

If we materially change how GatherOS handles data, we'll update this document, bump the "Last updated" date, and surface the change in the app's What's New release notes. The current version of this policy is always available inside the app at **Settings → About → Privacy policy**, and on GitHub at [`PRIVACY.md`](./PRIVACY.md).
