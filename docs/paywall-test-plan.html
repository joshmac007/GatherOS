Paywall / trial — test plan

Work top to bottom. Each step tells you where to type (Terminal = the macOS Terminal app, DevTools = the app's developer console) and exactly what to paste. Tick things off as you go — progress is saved in this browser.

The 3 modes you're testingMeans
paidactive subscription → everything works, no nags.
trial14-day free trial running (no card) → everything works.
freetrial over + no subscription → old saves still work, new saves & pro features ask you to upgrade.

★ Before you start do this once

Open the Terminal

Press ⌘ Space, type Terminal, hit Return. A text window opens — that's where any Terminal command goes (paste, then press Return).

Launch the app

Terminal — paste this and leave it running (it opens GatherOS):

cd /Users/brett/GatherOS && git checkout main && git pull origin main && npm run dev

To quit the app later: click the GatherOS window and press ⌘ Q. To relaunch, run the line above again.

Open the app's DevTools console

Click the GatherOS window, press ⌥ ⌘ I (Option-Command-I). A panel opens — click the Console tab at the top. That's where any DevTools command goes (paste, press Return).

The one command you'll use constantly

DevTools — this prints the current mode. It's your source of truth in every step:

await window.moodmark.entitlement.get()
Reads back something like { mode: "trial", trial: { daysLeft: 14, active: true }, … }. The mode field is what you check.
Backend is already done. You set the Lemon Squeezy trial to 0 and deployed the worker with no server trial — so this plan only tests the app itself. Nothing here changes your server.

2 · Trial countdown, then free tier no account needed

This rewinds the local trial clock so you can see each stage. Each one is the same 3 moves: quit → paste one line → relaunch → check.

A · A fresh 14-day trial

  1. Quit the app (⌘ Q).
  2. Terminal — paste:
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"; node -e "const f='$APP/prefs.json',fs=require('fs');const p=JSON.parse(fs.readFileSync(f));p.trialStartedAt=Date.now();fs.writeFileSync(f,JSON.stringify(p,null,2));console.log('OK — fresh trial')"
  1. Relaunch (npm run dev), open DevTools, run await window.moodmark.entitlement.get().
Pass: mode: "trial", daysLeft: 14. No bottom banner yet.

B · Trial almost over (2 days left)

Quit, paste, relaunch, check:

APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"; node -e "const f='$APP/prefs.json',fs=require('fs');const p=JSON.parse(fs.readFileSync(f));p.trialStartedAt=Date.now()-12*864e5;fs.writeFileSync(f,JSON.stringify(p,null,2));console.log('OK — 2 days left')"
Pass: mode: "trial", and a small "Trial: 2 days left" nudge appears at the bottom (dismissible).

C · Trial over → free tier

Quit, paste, relaunch, check:

APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"; node -e "const f='$APP/prefs.json',fs=require('fs');const p=JSON.parse(fs.readFileSync(f));p.trialStartedAt=Date.now()-16*864e5;fs.writeFileSync(f,JSON.stringify(p,null,2));console.log('OK — trial expired')"
Pass: mode: "free", and the "You're on the free plan" banner is at the bottom.

3 · Don't punish current users the important one

Proves that someone updating to this version keeps their library and does not get a surprise new trial.

  1. Make sure you have a few saves in your library (you do). Quit the app (⌘ Q).
  2. Terminal — this clears the trial decision so the app re-decides on next launch, exactly like a real update would:
APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"; node -e "const f='$APP/prefs.json',fs=require('fs');const p=JSON.parse(fs.readFileSync(f));delete p.trialStartedAt;fs.writeFileSync(f,JSON.stringify(p,null,2));console.log('OK — simulating an update')"
  1. Relaunch, open DevTools, run await window.moodmark.entitlement.get().
Pass: trial.startedAt is 0 (the trial is counted as already used — no free 14-day reset). If you're not signed in with a paid plan, mode is "free".

6 · Never block a payer safety

The app should fail open — when unsure, it lets you in. Test while signed in as paid (or with a recent paid cache).

Terminal — corrupt the license cache (quit the app first):

APP="$(find ~/Library/Application\ Support -maxdepth 2 -name prefs.json | head -1 | xargs dirname)"; echo '{}' > "$APP/license-cache.json"; echo "OK — cache corrupted"
